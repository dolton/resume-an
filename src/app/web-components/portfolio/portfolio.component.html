Worked with a talented team of web developers to design, build, test and maintain the web application SwiftKanban. I learnt a lot while working on SwiftKanban as to how a enterprise level web application is built, maintained and used by other organisations. Build web services and perform integration with various tools for a better end using experience with feature rich application. SwiftKanban is a powerful, yet intuitive, Kanban software for helping you manage your Lean/ Agile initiatives. SwiftKanban combines powerful Visualization with Lean Analytics and Ease of Use to give you the best Team Kanban, Enterprise/ Portfolio Kanban and Scrum/ Scrumban capabilities.

While working in SwiftKanban and SwiftEase, there was a need to scale the application to handle higher load. To test the performance, I built two scripts using JMeter, the first one to test all the available web services and the second one for testing the performance of the basic operations of the application using a number of concurrent users who randomly performed the operations for a certain duration, normally one hour. The reports generated were then analysed and database was tuned accordingly. Also the code was improved and caching and queueing was implemented for reducing the database queries.

During working on building the performance scripts and improving the application performance, there was a need for a stable set of environment to get consistent results and improve. To achieve this, I wrote a script using a tool called Vagrant, which created virtual machines. Using the tool, the scripts were developed to create entire infrastructure on AWS using the ec2 containers. The machines were configured using shell scripting, which finally create a web infrastructure consisting of a loadbalancer, web servers, database server, search/ push server, jmeter slave machines and jmeter master machine. The entire infrastructure could be started and terminated on a single command to reduce the cost.
